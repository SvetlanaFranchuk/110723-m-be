1 уровень сложности: 1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?
git checkout <имя файла>

2.Как вернуть файл из индекса в рабочий каталог?
git restore --staged 
git reset (устаревший вариант)

3.Как отменить ошибочный commit ?
git reset HEAD~1 - перейти на 1 коммит назад, вместо 1 можно ставить кол-во коммитов на которое нужно вернуться

4.Как исправить коммит отправленный на сервер GitHub?
git revert HEAD - отменить последний коммит отправленный на сервер 

5.Как создать новую ветку и переключится в нее?
git branch <название ветки> - создать ветку
git checkout <название ветки> - переключиться на нее
git checkout -b <название ветки> создать ветку и сразу переключиться на нее
 
5.В чем отличение между fast-forward merge  and non fast-forfard merge?
1) слияние происходит "линейно", указатель главной ветки просто сдвигается
2) слияние происходит путем суммирования того, что происходило в разных ветках. гит создает новый дополнительный коммит

6.Когда может возникнуть конфликт слияния веток, как  его разрешить?
когда меняют один и тот же файл в разных ветках 

7.Можно ли удалить ветку? Как? 
можно. нужно сначала с нее перейти на другую (или на основную) git checkout <имя ветки> 
и потом выполнить команду git branch -d <имя ветки>
Плюс в этой ветке не должно быть незаконченной работы (не закомиченной). Можно использовать ключ -D
